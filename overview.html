Overview

What is GitHub? [1]

  • A cloud-based version control and collaboration platform built around Git.
  • Hosts repositories (repos) where developers store and manage code.
  • Enables team collaboration through features like pull requests, issues, and code reviews

KEY FEATURES

  1. Version Control (Git) - a) Tracks changes in code over time.
    - b) Allows reverting to previous versions.
  2. Repositories (Repos)- a) Contains project files, documentation, and version history.
    -b) Can be public (open-source) or private.
  3. Branching & Merging- a) Work on separate branches without affecting the main code (main/master).
    - a) Merge changes via pull requests (PRs).
  4. Collaboration Tools- a) Issues: Track bugs and tasks.
    - b) Pull Requests: Propose and review code changes.
    - c) Projects: Kanban-style task management.
  5. GitHub Actions- a)Automate workflows (CI/CD pipelines). .
  6. GitHub Pages- a)

BASIC COMMANDS

Command Description
git clone (url) Copy a repo to your local machine/PC
git add (file) Stage changes for commit
commit (message) Save changes with a description
git push Upload changes to GitHub
git pull Download changes from GitHub
